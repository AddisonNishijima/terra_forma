DESIGN CASE STUDY

01.OVERVIEW

The web application "Terra Forma" takes place in the future after the terraformation of Mars. The purpose of Terra Forma is to create the ability for colonists to build a lively community. Challenge.

02.CHALLENGE

The challenge includes that humans moving to Mars need infrastructure. They will need resources including: housing, government, entertainment, water, food, oxygen, and soil. Obtaining these elements will create a Utopia that has been invisioned and virtually built by these colonists. Process.

03.PROCESS

The first step of invisioning this project included appointing a Project Manager to break down the concepts of the steps that needed to be taken in order to start the designing process. As a group we brainstormed a vision for a future Mars by creating a color palette that consisted of futuristic, organic colors rather than the current state which is red and dusty. Once a vision board and color palette was in place, we had a foundation to start on. Each member of the group was appointed a task to work on that best fit their talents as a designer. From there, the pieces of the project started to come together. The sequence included: a survey to our fellow peers, creating low-fidelity wireframes that moved into medium-fidelity mock ups, which eventually turned into a full functioning work flow. Icons and other elements were created throughout the process and placed in the project appropriately.Solution.

SOLUTION

The main objections included a method for colonists to build a community. We obtained this by builing a map of Mars that consisted of 4 different cities that when hovered over, a magnified image popped up to show a more accurate image of the location. We used actually photos and exact coordinates to create a realistic web application. Results.
